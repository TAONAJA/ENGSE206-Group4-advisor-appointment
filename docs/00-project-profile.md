## 2. Elevator Pitch

> ใน 2–3 ประโยค อธิบายว่า “ระบบนี้ช่วยใคร แก้ปัญหาอะไร และสร้างคุณค่าอย่างไร”
 ระบบนัดหมายเข้าพบอาจารย์ที่ปรึกษาช่วยลดความซับซ้อนในการติดต่อโดยรวม ตารางเวลาของอาจารย์และสถานะนัดหมายของนักศึกษามาไว้ในระบบส่วนกลางแบบเรียลไทม์ โดยที่ระบบนี้แก้ปัญหานัดเวลาทับซ้อนและลดภาระของอาจารย์ที่ต้องคอยตอบคำถามเดิมซ้ำๆ เกี่ยวกับเวลาว่างในแต่ละวัน ทำให้การเข้าพบที่รวดเร็วและตรวจสอบสถานะได้ชัดเจน ช่วยให้จัดสรรเวลาได้อย่างมีประสิทธิภาพ ส่งผลให้การดำเนินงานวิจัยหรือโปรเจกต์ของนักศึกษาราบรื่นยิ่งขึ้น



## 3. Scope ระดับสูง

### In Scope
- กรณีการนัดหมายเข้าพบอาจารย์ที่ปรึกษาในภาควิชา:
ตั้งแต่ผู้ใช้ (นักศึกษา) ค้นหาสถานะ/เวลาว่างของอาจารย์ → ส่งคำขอนัดหมายพร้อมระบุเหตุผล → อาจารย์พิจารณา (อนุมัติ/ปฏิเสธ/เลื่อนนัด) → ผู้ใช้ทราบผลและรับการแจ้งเตือนเงื่อนไข


### Out of Scope
- ระบบลาเรียน/ลาป่วยอย่างเป็นทางการ; การนัดหมายอาจารย์ข้ามคณะ; การเชื่อมโยงกับระบบทะเบียนหลักของมหาวิทยาลัยที่มีความซับซ้อนสูง

## 4. ผู้ใช้หลัก

| Role | เป้าหมายหลัก | สิ่งที่คาดว่าจะทำกับระบบ |
|---|---|---|
| นักศึกษา / หัวหน้ากลุ่ม | - ต้องการทราบช่วงเวลาว่างของอาจารย์ที่ปรึกษาที่แน่นอน<br>- สามารถขอเนัดหมายเข้าพบล่วงหน้าได้สะดวก | เกิดความล่าช้า หรือเวลาไม่ตรงกับอาจารย์ที่ปรึกษา  |
| เจ้าหน้าที่ / ผู้ดูแลระบบ | -  เห็นข้อมูลคำขอนัดหมายและเหตุผลการเข้าพบครบถ้วน<br>- จัดการ ยืนยัน หรือเลื่อนตารางนัดหมายได้ง่าย
 | ต้องคอยตอบคำถามเดิมๆ เรื่องเวลาว่างซ้ำหลายครั้ง  |
| อาจารย์ / ผู้ดูแลพื้นที่ | ช่วยอำนวยความสะดวกเมื่ออาจารย์ติดภารกิจด่วน | ตารางนัดหมายอาจสะดุดหรือทับซ้อนกับเวลาของกิจกรรมหรือภาควิชา |

## 5. Success Criteria ระดับสูง

- [SC-01] ผู้ใช้สามารถทำทราบช่วงเวลาว่างของอาจารย์ที่ปรึกษาที่แน่นอน และสามารถขอนัดหมายเข้าพบล่วงหน้าได้สะดวก
- [SC-02] เจ้าหน้าที่/ผู้ดูแลระบ สามารถจัดการแก้ไข ยืนยัน หรือเลื่อนตารางนัดหมายได้ง่าย
- [SC-03] อาจารย์เห็นการนัดหมายและสามารถจัดระเบียบคิว บริหารจัดการตารางเวลา และสื่อสารเวลานัดหมายได้อย่างโปร่งใสและมีประสิทธิภาพมากขึ้น
